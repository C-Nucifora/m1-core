# m1-core v2 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Every task ends Red → Green → `cargo test` → commit.

**Goal:** Extend the v1 `m1-core` library (`parse`, `Cst`, `Node`, generated `Kind`, `Diagnostic`/`Range`/`Position`/`Severity`/`Code`) with grammar-driven ergonomics its four consumers actually need: a generated `Field` enum + `Node::child_by_field`, allocation-free traversal iterators (`child_nodes`/`named_child_nodes`/`descendants`), sibling navigation (`next_sibling`/`prev_sibling`), node-by-byte-offset lookup on `Cst` (`node_at_offset`/`named_node_at_offset`), and `Diagnostic`/`Node` construction helpers.

**Architecture:** `m1-core` is a Rust library wrapping `tree-sitter` + `tree-sitter-m1` (path dep); downstream tools see only m1-core types and never import `tree-sitter`. The `Kind` enum is generated from `tree-sitter-m1`'s `node-types.json` by the `xtask` workspace member, committed as `src/kind.rs`, with a freshness test. v2 adds a second generated file `src/field.rs` (the `Field` enum) from the same JSON, guarded by a matching freshness test. New traversal/accessor methods live in `cst.rs` (the only module touching `tree_sitter`); new diagnostic constructors live in the dependency-free `diagnostic.rs`. No new dependency; `serde_json` stays in `xtask`.

**Tech Stack:** Rust (edition 2021), `tree-sitter` 0.25 (behind the boundary, not re-exported), `tree-sitter-m1` (path), `serde_json` (in `xtask` only).

**Spec:** `docs/superpowers/specs/2026-05-31-m1-core-v2-design.md`

```rust
/// Produce the full contents of `src/field.rs` from a node-types.json string.
/// Field names are always grammar identifiers (snake_case), so this reuses
/// `pascal_case` and needs no symbol map.
pub fn generate_field_rs(node_types_json: &str) -> String {
    let parsed: serde_json::Value =
        serde_json::from_str(node_types_json).expect("node-types.json must be valid JSON");
    let entries = parsed.as_array().expect("node-types.json must be an array");

    // Distinct field-name strings across all node types, sorted for determinism.
    let mut fields: BTreeMap<String, ()> = BTreeMap::new();
    for e in entries {
        if let Some(obj) = e.get("fields").and_then(|v| v.as_object()) {
            for name in obj.keys() {
                fields.insert(name.clone(), ());
            }
        }
    }

    // (variant_name, field_string), sorted by variant name; detect collisions.
    let mut variants: Vec<(String, String)> = fields
        .keys()
        .map(|name| (pascal_case(name), name.clone()))
        .collect();
    variants.sort();
    for pair in variants.windows(2) {
        assert!(
            pair[0].0 != pair[1].0,
            "two field names map to variant {:?}: {:?} and {:?}",
            pair[0].0, pair[0].1, pair[1].1
        );
    }

    let mut out = String::new();
    out.push_str("// @generated by `cargo run -p xtask -- gen-kinds`. Do not edit by hand.\n");
    out.push_str("//! Typed node fields for the M1 grammar, generated from node-types.json.\n\n");
    out.push_str("#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]\n");
    out.push_str("pub enum Field {\n");
    for (variant, _) in &variants {
        out.push_str(&format!("    {variant},\n"));
    }
    out.push_str("}\n\n");
    out.push_str("impl Field {\n");
    out.push_str("    /// The grammar field-name string, for tree-sitter lookups.\n");
    out.push_str("    pub fn as_str(self) -> &'static str {\n");
    out.push_str("        match self {\n");
    for (variant, name) in &variants {
        out.push_str(&format!("            Field::{variant} => {name:?},\n"));
    }
    out.push_str("        }\n");
    out.push_str("    }\n");
    out.push_str("}\n");
    out
}

fn field_rs_path() -> PathBuf {
    Path::new(env!("CARGO_MANIFEST_DIR")).join("../src/field.rs")
}
```

```rust
/// Iterator over a node's direct children, yielded lazily. When `named_only`
/// is set, only named children are visited. Allocates nothing per element.
pub struct Children<'a> {
    parent: tree_sitter::Node<'a>,
    source: &'a str,
    index: usize,
    count: usize,
    named_only: bool,
}

impl<'a> Iterator for Children<'a> {
    type Item = Node<'a>;

    fn next(&mut self) -> Option<Node<'a>> {
        while self.index < self.count {
            let i = self.index;
            self.index += 1;
            let child = if self.named_only {
                self.parent.named_child(i)
            } else {
                self.parent.child(i)
            };
            if let Some(inner) = child {
                return Some(Node {
                    inner,
                    source: self.source,
                });
            }
        }
        None
    }
}
```

```rust
        let cst = parse("x = a + b;\n");

        // Reference: recursive children() walk (node first, then each subtree).
        fn rec<'a>(n: crate::Node<'a>, out: &mut Vec<(Kind, std::ops::Range<usize>)>) {
```

```rust
/// Pre-order iterator over a node and all of its descendants (node first, then
/// each child's subtree, left to right). Uses a single worklist for the whole
/// traversal rather than allocating a child vector per node.
pub struct Descendants<'a> {
    stack: Vec<tree_sitter::Node<'a>>,
    source: &'a str,
}

impl<'a> Iterator for Descendants<'a> {
    type Item = Node<'a>;

    fn next(&mut self) -> Option<Node<'a>> {
        let inner = self.stack.pop()?;
        // Push children in reverse so the leftmost is popped next (pre-order).
        let count = inner.child_count();
        for i in (0..count).rev() {
            if let Some(child) = inner.child(i) {
                self.stack.push(child);
            }
        }
        Some(Node {
            inner,
            source: self.source,
        })
    }
}
```

Then add the method inside `impl<'a> Node<'a>` (after `named_child_nodes`):

```rust
    pub fn descendants(&self) -> Descendants<'a> {
        Descendants {
            stack: vec![self.inner],
            source: self.source,
        }
    }
```

```rust
    /// The smallest node whose byte span contains `offset` (any node). Offsets
    /// past end-of-file clamp to the document length; always returns a node.
    pub fn node_at_offset(&self, offset: usize) -> Node<'_> {
        let off = offset.min(self.source.len());
        let inner = self
            .tree
            .root_node()
            .descendant_for_byte_range(off, off)
            .unwrap_or_else(|| self.tree.root_node());
        Node {
            inner,
            source: &self.source,
        }
    }

    /// The smallest *named* node whose byte span contains `offset`. Offsets past
    /// end-of-file clamp; always returns a node.
    pub fn named_node_at_offset(&self, offset: usize) -> Node<'_> {
        let off = offset.min(self.source.len());
        let inner = self
            .tree
            .root_node()
            .named_descendant_for_byte_range(off, off)
            .unwrap_or_else(|| self.tree.root_node());
        Node {
            inner,
            source: &self.source,
        }
    }
```

```rust
#[test]
fn corpus_traversal_is_faithful() {
    let dir = corpus_dir();
    if !dir.is_dir() {
        eprintln!("corpus dir not found ({}); skipping", dir.display());
        return;
    }

    let mut checked = 0usize;
    for entry in std::fs::read_dir(&dir).expect("read corpus dir") {
        let path = entry.expect("dir entry").path();
        if path.extension().and_then(|e| e.to_str()) != Some("m1scr") {
            continue;
        }
        let src = std::fs::read_to_string(&path).expect("read script");
        let cst = m1_core::parse(&src);

        // descendants() must equal a recursive children() pre-order walk.
        fn rec(n: m1_core::Node, out: &mut Vec<std::ops::Range<usize>>) {
            out.push(n.byte_range());
            for c in n.children() {
                rec(c, out);
            }
        }
        let mut reference = Vec::new();
        rec(cst.root(), &mut reference);
        let via_iter: Vec<_> = cst.root().descendants().map(|n| n.byte_range()).collect();
        assert_eq!(
            via_iter, reference,
            "descendants() diverged from recursive walk in {}",
            path.file_name().unwrap().to_string_lossy()
        );

        // node_at_offset must be total: sample offsets across the file.
        let step = (src.len() / 50).max(1);
        let mut off = 0;
        while off <= src.len() {
            let _ = cst.node_at_offset(off).kind();
            let _ = cst.named_node_at_offset(off).kind();
            off += step;
        }

        checked += 1;
    }
    assert!(checked >= 80, "expected >= 80 corpus scripts, found {checked}");
}
```

## Deferred to v3

| Item | Why deferred |
|------|--------------|
| Incremental / edit reparse (`Tree::edit` + reuse old tree) | No consumer needs it yet; `m1-lsp` re-parses whole documents fast enough. Exposing `InputEdit` cleanly (without leaking `tree_sitter`) needs its own design. |
| `.m1prj` / `.m1cfg` symbol model + name resolution | Deferred since v1; `m1-typecheck` keeps a local model. Belongs in its own spec. |
| UTF-16 / LSP position encoding in `m1-core` | Stays in `m1-lsp`'s `LineIndex`; `m1-core` remains byte-oriented (v1 decision). |
| tree-sitter query (`.scm`) API surfaced through m1-core | No consumer needs pattern queries; `descendants()` + `Kind`/`Field` cover current traversal needs. |
| Typed per-kind wrapper structs (e.g. `BinaryExpression { left, op, right }`) | `Kind` + `Field` accessors deliver the ergonomics at far less surface area; revisit only if accessor churn proves painful. |
| Mutable / owned / red-green CST | Out of scope; the wrap-and-borrow model suffices. |
